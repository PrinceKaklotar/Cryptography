🛡️ Core Features

Why Hybrid Encryption?

Classical computers can't break LWE. Quantum computers can't efficiently break LWE. If one algorithm falls, the other stands. That's the power of hybrid cryptography.

🌐

Classical Security (ECDH)

Elliptic Curve Diffie-Hellman over P-256. Provides 128-bit classical security. Used in TLS, Signal, WhatsApp today.

128-bit classical P-256 curve
🔮

Post-Quantum Security (LWE)

Learning With Errors — a lattice-based problem believed to be hard even for quantum computers running Shor's algorithm.

Quantum-resistant NIST finalist
⚡

Hybrid Approach

Combine both shared secrets via SHA-256: Key = H(ECDH || LWE). Safe even if either algorithm is broken.

🧮 The Hard Problem

What Makes LWE Quantum-Hard?

Imagine you receive a table of random numbers (matrix A) and a vector b. You're told that b ≈ A·s for some secret s, but with a tiny random error added. Finding s — even knowing A and b — is believed to be computationally hard for both classical and quantum computers.

💡
The Analogy

It's like a noisy linear equation system. You have 4 equations, 4 unknowns — but each equation has a small error. Solving it exactly is easy, but the error randomizes every attempt. That randomness is what makes LWE hard.

1
Generate random A, small secret s

A is a public n×n matrix. s is a small secret vector.

2
Compute b = A·s + e

Add a tiny error e from a narrow distribution χ. This is the "Learning With Errors" sample.

3
Publish (A, b) as public key

Finding s from A and b is the LWE hard problem. Even quantum computers can't do it efficiently.

LWE Sample (Demo: n=4, q=97)
A = random matrix ∈ ℤ974×4
s = secret vector ∈ {0,1,2}4
e = error vector from χ, |eᵢ| ≤ 1

🛡️ Defense in Depth

Why Hybrid is Future-Proof

The hybrid approach ensures your data is safe today and tomorrow, regardless of which algorithm might eventually be weakened.

⚠️ Shor's Algorithm Breaks ECDH

A sufficiently powerful quantum computer running Shor's algorithm can solve the Elliptic Curve Discrete Logarithm Problem (ECDLP), breaking ECDH.

✓

LWE remains secure. The hybrid key is still protected.

⚠️ Future Lattice Attack on LWE

If a future mathematical breakthrough weakened LWE (e.g., better lattice reduction algorithms), quantum or classical, ECDH would still hold.

✓

ECDH remains secure. The hybrid key is still protected.

🔐

An attacker must break both

Since hybridKey = SHA-256(ECDH_secret || LWE_secret), recovering the key requires breaking both ECDH and LWE simultaneously. The probability of this is negligible under current cryptanalytic knowledge.
